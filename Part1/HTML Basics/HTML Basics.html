Lang

lang is used to describe the language of the page, the Chinese specification is zh-cmn-Hans, but also common zh-CN

Meta(required)

              <meta charset="UTF-8">
              <meta name="viewport" content="width=device-width, initial-scale=1.0">

charset: the character encoding set is a general term for various scripts and symbols, including various national scripts, punctuation marks, graphic symbols, numbers, etc. UTF-8 is usually used, which is the most comprehensive character set. There are many more, such as ASCII, etc.

viewport: the visible window size, is used to display that part of the web page

content: content is set to ensure the best display

These are the two metadata that must be included in the HTML base framework

Meta(optional)

              <meta name="keywords" content="HTML, CSS">
              <meta name="desription" content="...">
              more...

keywords: commonly used in the past to provide keyword indexing to search engines, but now is obsolete, but still available

description: used to describe the content of the page, the search engine will be displayed in its pages according to the content provided

Hyperlinks

              <a herf="http://google.com/" target="_blank"></a>
              <a herf="#section-1"></a>
              <h1 id="section-1">
                  Content
              </h1>

a: Anchor, tag can be a hyperlink or an anchor

herf: Hypertext Reference, means the URL specifying the target of the hyperlink. the value of the href attribute can be the relative or absolute URL of any valid document, including fragment identifiers and JavaScript snippets

relative path: .../coffee

absolute path: fancyshawn.com/coffee

target: usually followed by a tab in the jump page, with the content entered as _blank meaning open a new tab

Anchor point: used for positioning within the page fixed point (click on the corresponding position automatically slide down), first have to create an anchor point, select a tag to be positioned, give id="section-name", and then in a tag enter #section-name that can be. (If left blank with only #, then jump back to the top of the page)

Images

<img src="image/coffee.jpg" alt="A coffee mug on a table." style="object-fit: cover"/>

img: image tags generally use self-closing < / >, nowadays the syntax can be written without self-closing. (But it seems that when writing React, you still have to write it, so get used to it)

src: the image source, fill in the relative or absolute path

alt: Alter, text description, used to display when the image can not be loaded correctly, but also to help search engines understand what we put the image here is, it is highly recommended to write meaningful content

object-fit: one of the CSS styles, fill in cover to keep the image from being stretched and distorted

Strcturing a Web Page

The header tag is usually used to introduce the structure of a web page, and is usually used for the navigation bar

The nav tag is usually used for the navigation bar

The main tag is usually used to contain the main content of the page, and a page can only have one main tag, otherwise it will report an error during the specification validation

The section tag is usually used to sort and manage the related content. The heading tag is required to use section, otherwise an error will be reported when going to the specification validation

The aside tag is usually used to distinguish content that is not directly related to the content in main

The footer tag is usually used to contain information such as copyright, collaboration, and filing
